Roadmap

The following represents current view of Alien4Cloud development team of its product development cycle and future directions. It is intended for information purposes only, and should not be interpreted as a commitment, though we do our best to reach the dates and features set mentioned below.

Premium features, support and certifications are available under a subscription, delivered by FastConnect, a Bull, Atos technologies subsidiary. Contact us for more details.

We deliver major releases every 4 month. Please note that we also deliver intermediate sprint releases in Alien4Cloud GitHub repo, every 3 weeks, in order to get feedback on features while still in development.

  • July 2016 v 1.2.0

    Core

    • API versioning
    • Catalog Improvements
      • Portability insights - Premium
      • Sort components by date
    • UI Improvements
      • Location Resource Configuration: Provide suggestions for compute node configuration (os type, distribution, architecture, versions)
      • Location Resource Configuration: Improve visibility of incomplete configuration of resources
      • Catalog views: Add last import date and import source (git, manual, ...) in CSAR list
      • Design view: Ability to display operation implementation scripts directly from workflow view
      • Design view: Improve visibility of mandatory properties in topology design view
      • Design view: Add a URL link towards type containing the relationships in design view
      • Application list view: Display last launch time per environment.
      • Application navigation: Keep environnement context when switching from a screen to another
      • Runtime view: Display the deployment location
      • Runtime view: Prevent from changing scaling when scaling is already in progress
      • Runtime view: Display deployment errors informations from underlying orchestrator
    • Policies
      • Multi zones placement policies : ability to manage HA or topology performance by associating nodes to zones
      • Configure post deployment (runtime) policy on a topology
      • Support for host failure policy
    • Upgrade
      • Ability to upgrade an artefact managed by application infrastructure
      • Upgrade component content and version without updating relationships
    • Blueprints Developer experience improvements
    • Simplified Self Service
    • Security
      • SAML integration - Premium
      • Manage SSH Keys
      • Associate existing Security Groups to Compute

    Orchestrators certifications

    • Cloudify v3.3.1
    • Cloudify v3.4 (subject to cloudify roadmap)
    • Apache Brooklyn beta

    Infrastructure support

    • OpenStack Icehouse and Kilo
    • Amazon EC2 VPC support - Premium
    • Azure (ARM) support (linux and windows compute, volumes, public ips and network, security groups, H.A. policy)
    • VMWare VSphere 5.5 (compute, volumes, networks) - Premium
    • Docker

    TOSCA

    • Simple profile in YAML v1.0
    • Support for alien 4 cloud 1.1 DSL

    Additional Premium features

    • v1.1 to v1.2 Migration tools
  • To be defined Future
