Capabilities

A Capability Type is a reusable entity that describes a kind of capability that a Node Type can declare to expose. Requirements (implicit or explicit) that are declared as part of one node can be matched to (i.e., fulfilled by) the Capabilities declared by other node.

Keynames

Keyname Type Required Description
derived_from string no Name of the capability type from which the defined capability type derives (i.e. itâ€™s parent type).
documentation string no Description of the capability.
properties list of property definitions. no Optional list of properties that the capability type defines.

It is recommended when adding a new capability to derive from the normative tosca.capabilities.Feature capability or another normative capability.

Grammar

<capability_type_name>:
  derived_from: <parent_capability_type_name>
  documentation: <documentation>
  properties:
    <property_definitions>

Documentation on property definitions can be found here.

Example

Example of the definition of a capability

capability_types:
  fastconnect.capabilites.CapabilitySample:
    derived_from: tosca.capabilities.Feature
    documentation: A sample capability type.
    properties:
      property_1:
        type: string
